# Profit Proof Thesis

Ironflyer becomes valuable only if it proves three things together:

```text
Paid demand
+
Gross margin
+
Completion reliability
```

A system that completes projects but loses money is not scalable.
A system that earns money but fails projects loses retention.
A system that has users but no cost control dies from inference and runtime bills.

## Required Commercial Proof

1. User pays before runtime burns significant cost.
2. Runtime cost is tracked per execution.
3. Profit Guard blocks unprofitable paths.
4. Completion score proves value delivered.
5. Users refill or upgrade after seeing previews/results.
6. Gross margin remains positive under load.
7. Scale does not increase cost linearly with expensive reasoning.

## Proof Formula

```text
Commercial Proof =
paid_executions
* completion_rate
* gross_margin
* repeat_usage
```

## What Counts As Real Proof

Not:
- signups
- demos
- generated code
- screenshots
- architecture documents

Yes:
- paid wallet top-ups
- paid executions
- preview delivered
- completion report delivered
- margin retained
- repeat usage
- low refund rate
