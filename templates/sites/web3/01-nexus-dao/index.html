How it works

Propose, vote, execute.

Three signed steps from a forum thread to an onchain transaction. The timelock is mandatory and the verdict is on a block explorer before the queue moves.

01 / propose

Draft and pin a proposal

Submit a proposal with calldata, target contracts, and value. Bond a small NXS deposit. Discussion happens onchain, signed by the proposer's wallet.

02 / vote

Gasless tally with onchain settle

Voters sign EIP-712 messages on Snapshot. When quorum and majority are reached, the tally is settled onchain in a single transaction, ready for the queue.

03 / execute

Timelocked multisig execution

Approved proposals enter a 48-hour timelock. A 5-of-9 multisig signs the execution. Every step is on-chain, in your block explorer, before the funds move.

Modules

The governance stack, composable.

Mix and match: treasury, voting, delegation, and execution modules. Swap implementations without redeploying the DAO.

T

Treasury module

Multi-asset vault with daily spend limits, allowlisted recipients, and onchain accounting. Integrates with Safe and Zodiac roles.

V

Voting module

ERC-20 weighted, NFT-gated, or quadratic. Snapshot integration for gasless polls, onchain Bravo for binding votes.

D

Delegation

EIP-5805 vote delegation with checkpoints. Track delegate performance, revoke instantly, attest your delegate's voting record.

L

Timelock

OpenZeppelin TimelockController under the hood. Configurable delay per action class, emergency veto multisig, full event log.

M

Multi-chain

Run treasuries on Ethereum, Optimism, Arbitrum, Base, and Polygon. Cross-chain calls via CCIP and LayerZero, settlement on mainnet.

A

Attestations

EAS-backed proposal attestations. Voters, delegates, and execution multisigs leave a signed, queryable record on every action.

Developers

SDK first. Forge-ready.

TypeScript SDK, Foundry templates, and Hardhat plugins. Deploy a governance stack with sane defaults, then peel back to raw calldata when you need it.

scripts/deploy.ts 
// deploy a full governance stack with token + timelock + voting
import { NexusFactory } from "@nexus-dao/sdk";

const stack = await NexusFactory.deploy({
  chain: "optimism",
  token: { name: "Field", symbol: "FIELD", supply: 100_000_000n },
  quorum: 4,                // percent of total supply
  timelock: 48 * 60 * 60, // seconds
  modules: ["treasury", "voting-bravo", "delegation"],
});

console.log(stack.governor, stack.timelock, stack.token);
// ✓ deployed at block 121,448,302 · gas 4.2M · cost $34.18
